# v0.2.1 Final Report / PR Description

## Summary

- Added v0.2.1 shadow batch replay over resolved BTC 5-minute markets, reusing the shared `buildMarketState`, `generateSignal`, and `labelSignalOutcome` strategy functions.
- Added idempotent persistence support with unique indexes on `market_states (market_id, ts)` and `signals (market_state_id)`, plus `insertIfAbsent` repository helpers.
- Added a signal-density report with per-bucket `total`, `accepted`, `labeledAccepted`, `win`, and `loss` counters; `labeledAccepted` is the sample-size gate.
- Fixed migration tooling so `pnpm db:migrate` and `pnpm --filter @pulse5/storage migrate:create` work with pnpm's virtual-store layout.

## Real-Data Dry-Run

Source database: local `pulse5tester` capture data.

The tester database originally had unresolved markets only, so outcomes were backfilled for test-only replay using Chainlink start/end ticks when both were within 10 seconds of the market boundary. Preview showed 340 backfillable markets with max start/end delta of 3000 ms.

Command:

```powershell
$env:DATABASE_URL = "postgres://postgres:postgres@127.0.0.1:5432/pulse5tester"

npx tsx research/replay/replay-batch.ts `
  --from=2026-04-25T08:00:00Z `
  --to=2026-04-26T23:30:00Z `
  --limit=200 `
  --step-ms=5000 `
  --report=research/reports/v0.2.1-density-pulse5tester-limit200-dry-run.json
```

Result:

```text
markets.observed = 200
markets.replayReady = 187
markets.skipped = 13
states.built = 11033
signals.total = 11033
signals.accepted = 0
signals.acceptedRate = 0
```

Main rejection reasons:

```text
NO_EDGE = 11018
BTC_TOO_CLOSE_TO_PRICE_TO_BEAT = 10753
ENTRY_PRICE_TOO_EXPENSIVE = 4346
TIME_REMAINING_TOO_HIGH = 2057
```

Interpretation:

- The batch replay pipeline works against real captured data.
- Chainlink `price_to_beat` fallback worked: `PRICE_TO_BEAT_MISSING = 0`.
- Data freshness was not the blocker: stale/data-incomplete counts were near zero.
- The current v0.2 shadow engine produced zero accepted signals in this replay window, so there are no labeled accepted samples for calibration.
